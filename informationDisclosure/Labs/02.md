## [Information disclosure on debug page](https://portswigger.net/web-security/information-disclosure/exploiting/lab-infoleak-on-debug-page)

## 1. الهدف
استخراج قيمة متغير البيئة `SECRET_KEY` من صفحة التصحيح المعرّضة.
## 2. الخطوات 

1. افتح صفحة الموقع الرئيسية وافتح كود المصدر ب  (`Ctrl+U`) و لاحظ وجود تعليق يشير إلى صفحة تصحيح مثل:
 `<!-- <a href=/cgi-bin/phpinfo.php>Debug</a> -->`.
    
2. افتح العنوان مباشرة في المتصفح:
 `https://<lab-domain>/cgi-bin/phpinfo.php`.
    
3. افحص مصدر الصفحة أو استخدم البحث داخل الصفحة (`Ctrl+F`) وابحث عن `SECRET_KEY`.
    
4. ستجد سطرًا مشابهًا:
    
    ```html
    <tr><td class="e">$_SERVER['SECRET_KEY']</td><td class="v">28ca2kkbrn02d6ezjsupqdw6locqng1o</td></tr>
    ```
    
5. القيمة المستخرجة لـ `SECRET_KEY` هي: `28ca2kkbrn02d6ezjsupqdw6locqng1o`.
    

---

## 4. التحليل

- **نوع الثغرة**: Information disclosure عبر صفحة تصحيح مكشوفة.
    
- **السبب**: وجود صفحة تصحيح (`phpinfo`) متاحة للعامة، أو ترك رابط/تعليق يقود إلى هذه الصفحة.
    
- **التأثير**: تعرض متغيرات البيئة الحساسة مثل مفاتيح سرية وبيانات اعتماد قد يؤدي إلى استغلالات لاحقة.
