# [Source code disclosure via backup files](https://portswigger.net/web-security/information-disclosure/exploiting/lab-infoleak-via-backup-files)

**الهدف**  
استخراج كلمة مرور قاعدة البيانات من الشيفرة المُسرّبة في ملف احتياطي داخل مجلد مخفي.

**النتيجة**  
تم العثور على كلمة المرور التالية في الشيفرة المسربة:

`3esxqoe234wfix3pf5x2snshjbxglb6q`

---
## الخطوات 

1. بصيت علي ملف  `robots.txt` و لقيت  سطرًا يشير إلى  :  
    `Disallow: /backup`
    
2. فتحت المجلد `/backup` في المتصفح ووجدت قائمة الملفات (directory listing).
    
3. دخّلت على الملف الاحتياطي `ProductTemplate.java.bak`.
    
4. بحثت داخل الملف عن سلاسل اتصال أو متغيرات البيئة فوجدت جزء الكود التالي:
    

```java
ConnectionBuilder connectionBuilder = ConnectionBuilder.from(
    "org.postgresql.Driver",
    "postgresql",
    "localhost",
    5432,
    "postgres",
    "postgres",
    "3esxqoe234wfix3pf5x2snshjbxglb6q"
).withAutoCommit();
```

5. قيمة الحقل الأخير هي كلمة المرور لقاعدة البيانات.
---

## 3. التحليل والتأثير

- **نوع الثغرة**: Source code disclosure via backup files (Information Disclosure).
    
- **السبب**: نسخ احتياطية أو ملفات مؤقتة مُحْمَلة إلى خادم الويب، أو تُركت في مجلد عام، وربما تمّ تضمين بيانات اعتماد داخلها.
    
- **التأثير**: يمكن للمهاجم استخدام كلمة المرور للاتصال بقاعدة البيانات محليًا أو عبر الشبكة إن أمكن الوصول. هذا قد يؤدي لتسريب بيانات، تعديلها، أو تنفيذ عمليات ضارة أخرى.
